Separate concerns clearly (models, storage, business logic, CLI interface) to enable smooth evolution in later phases. Maintain clear boundaries between different layers of the application to support future extensibility.

### III. Simplicity and Readability
Prioritize straightforward, idiomatic Python code that is easy to understand and extend. Use meaningful variable/function names, comprehensive docstrings, and type hints for public interfaces to enhance maintainability.

### IV. Reliability and User Experience
Implement graceful handling of user input, clear feedback, and robust error management in the console. Ensure the application provides helpful error messages and intuitive user interactions.

### V. Future-Proof Design
Structure the code and project layout to support seamless progression to web, database, AI agents, and cloud-native phases. Design with extensibility in mind while maintaining current functionality.

Use Python standard library only for core functionality (no external packages required). This ensures portability, reduces dependencies, and simplifies deployment and maintenance.
